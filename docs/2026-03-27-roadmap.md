# acpx Roadmap

Prioritized plan for stabilizing, polishing, and expanding acpx from its current v0.1.3 baseline.

## Phase 1: Stabilize

Triage critical bugs and merge pending fixes that address user-reported pain.

| Item    | Type | Description                                                                              |
| ------- | ---- | ---------------------------------------------------------------------------------------- |
| PR #173 | fix  | Enable `shell: true` for terminal spawn on Windows (fixes #169)                          |
| PR #172 | fix  | Skip `session/load` for pristine sessions (fixes #165 first-event delay)                 |
| PR #164 | fix  | Auto-detect `CLAUDE_CODE_EXECUTABLE` when spawning Claude ACP agent                      |
| #184    | bug  | `protocolVersion` type mismatch with Gemini CLI (`--acp`)                                |
| #182    | bug  | Silent spawn failure after gateway restart — error swallowing leads to dead session loop |
| PR #168 | fix  | Add connection timeout and max buffer size limit                                         |

### Success criteria

- All merged fixes have passing tests.
- Gemini adapter initializes without type errors.
- Spawn failures surface actionable errors instead of silent loops.

## Phase 2: Platform and Polish

Harden cross-platform support, update dependencies, and improve error UX.

| Item    | Type  | Description                                                                 |
| ------- | ----- | --------------------------------------------------------------------------- |
| CI      | infra | Add Windows runner to test matrix                                           |
| PR #177 | chore | Merge dependabot dev-dependency updates                                     |
| #157    | bug   | `sessions history` returns "No history" for active session with live output |
| #176    | ux    | ACP session errors should include actionable remediation hints              |
| #151    | bug   | Inconsistent pinned version between package.json and runtime doctor/config  |

### Success criteria

- Tests pass on macOS and Windows CI.
- Dev dependencies are current.
- Error messages include remediation guidance.

## Phase 3: Feature Expansion

Add the most-requested capabilities.

| Item | Type | Description                                                   |
| ---- | ---- | ------------------------------------------------------------- |
| #49  | feat | `--model` and `--allowed-tools` flags for agent passthrough   |
| #148 | feat | `session/set_model` support (generic model selection via ACP) |
| #139 | feat | Per-tool permission policies with session-based escalation    |
| #134 | feat | Output format option to suppress file READ operations         |

### Success criteria

- Model passthrough works for codex and claude adapters.
- Permission policies can be scoped to individual tool names.
- `acpx <agent> sessions show` outputs session content.

Expand adapter support and enable advanced orchestration patterns.
